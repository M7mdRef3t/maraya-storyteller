# نقد تصميم مرايا — تقرير شامل
## Design Critique Report · Apple Design Manager Perspective

**المُقيِّم:** مدير تصميم، نهج Apple HIG + Nielsen Heuristics
**المُقيَّم:** مرايا — واجهة تطبيق سرد قصصي بالذكاء الاصطناعي
**المصادر:** 11-ux-design-spec.md · 10-brand-identity-book.md · App.css

---

## المقدمة

مرايا تصميم طموح يُجسّد رؤية سينمائية حقيقية. إذا أردتُ أن أصفه بجملة واحدة من منظور Apple:

> **"العاطفة صحيحة. بعض القرارات التنفيذية تحتاج جراحة."**

هذا النقد بنّاء — يُثمّن ما هو ممتاز ويُعالج ما هو قابل للتحسين بدقة. كل ملاحظة مرفقة بإصلاح قابل للتنفيذ.

## القسم الأول: هيوريستيات Nielsen العشر

### النظام التقييمي

```
5 ★★★★★ — مثالي، مرجعي
4 ★★★★☆ — جيد جداً مع هامش بسيط
3 ★★★☆☆ — مقبول، يحتاج تحسين
2 ★★☆☆☆ — مشكلة واضحة تؤثر على الاستخدام
1 ★☆☆☆☆ — حرج، يعطّل التجربة
```

### H1 — رؤية حالة النظام (Visibility of System Status)

**الدرجة: 4 / 5 ★★★★☆**

**ما يعمل بشكل ممتاز:**
- `LoadingOrb` مع رسائل دورية ("مرايا تتأمل...", "القصة تتشكّل...") → يُبقي المستخدم منخرطاً
- `SceneProgress` pill ("المشهد ٢ من ٥") → وضوح للتقدم الهرمي
- `AudioHUD` يُظهر الحالة بوضوح (ON/OFF + اللون الأخضر)

**ما يحتاج تحسين:**
- **خلال توليد الصورة:** لا يوجد مؤشر مُميَّز لتوليد الصورة تحديداً (فقط الـ Orb العام). المستخدم لا يعرف هل الذكاء الاصطناعي يكتب نصاً أم يولد صورة
- **أثناء الضغط على ChoiceButton:** الـ feedback البصري بين الضغط والانتقال للـ Loading ليس تآنياً كافياً (300ms فجوة)

**الإصلاح:**
```
اضف micro-indicator صغير تحت الـ Loading Orb:
• "يكتب القصة..." → Typewriter icon pulse
• "يرسم المشهد..." → Paintbrush icon pulse
→ يُعطي وعياً أعمق بما يجري
```

### H2 — التطابق مع العالم الحقيقي (Match Between System and Real World)

**الدرجة: 5 / 5 ★★★★★**

**ما يعمل بشكل ممتاز:**
- لغة "مرايا تتأمل"، "القصة تتشكّل"، "المرآة لا تنسى" → مُجانسة بديعة مع استعارة المرآة
- أسماء مشاعر مباشرة: فرح، حزن، غضب — لا مصطلحات تقنية
- أزرار الخيار بصيغة أفعال سردية: "افتح الباب الغامض" لا "اختيار 1"
- "توقف الخيط، نُعيد نسجه" بدلاً من "Error 503"

**ملاحظة المدير:**
> هذا الٌبعد هو التميّز الحقيقي لمرايا. الكتابة المتناسقة مع الاستعارة تُفرّق بين تطبيق يُشعرك براحة البيت وبين واجهة تجعلك وكأنك داخل برنامج. **لا تُغيّر هذا أبداً.**

---

### H3 — التحكم الحرية (User Control and Freedom)

**الدرجة: 3 / 5 ★★★☆☆**

**ما يعمل:**
- زر "رجوع" في Space Upload
- زر "قصة جديدة" في الختام

**مشاكل:**
1. **لا "تراجع" أثناء القصة:** المستخدم لا يستطيع "العودة خطوة" في المشهد السابق. بمجرد الاختيار، لا عودة.
2. **لا إيقاع مؤقت:** لا يمكن "إيقاف مؤقت" للقصة حين يريد المستخدم التفكير
3. **إذا انتهى الـ Onboarding وأراد إعادة المشاهدة:** لا توجد طريقة

**الإصلاح المقترح:**
```
أضف في شاشة القصة:
• زر ⟵ صغير جداً (glass pill) يُعيد المشهد السابق فقط
  "أقرأ من جديد" → يُعيد السرد بدون إعادة التوليد
• Haptic: .light + title: "العودة للمشهد السابق؟"
  Confirmation: [نعم] [لا، كمّل]
```

### H4 — الاتساق والمعايير (Consistency and Standards)

**الدرجة: 4 / 5 ★★★★☆**

**ما يعمل:**
- Glass surface متسقة عبر كل المكوّنات
- قواعد الألوان العاطفية واضحة ومتكررة
- typography hierarchy ثابتة

**مشاكل:**
1. **أحجام border-radius غير متسقة:**
   - ChoiceButton: `1.2rem` (19.2px)
   - EmotionCard: `1rem` (16px)
   - GlassCard: `1rem` (16px)
   - SceneProgress: `999px`
   - VoiceToggle: `999px`
   → 5 قيم مختلفة، بدون منطق واضح للاختيار بينها

2. **padding الأزرار غير موحّدة:**
   - AudioHUD btn: `0.45rem 0.7rem`
   - VoiceToggle: `0.42rem 0.7rem`
   - EmotionPickerUpload: `0.8rem 1.5rem`
   → بدون نظام tokens

**الإصلاح:**
```css
/* توحيد Radius System */
--radius-interactive: 12px; /* inputs, cards */
--radius-pill:        999px; /* badges, tags, pills */
--radius-button-md:   12px;
--radius-button-lg:   16px;
--radius-sheet:       20px 20px 0 0;

/* ثم طبّقها بشكل صارم في كل مكوّن */
```

### H5 — منع الأخطاء (Error Prevention)

**الدرجة: 3 / 5 ★★★☆☆**

**ما يعمل:**
- Dropzone تقبل فقط Image files — تصفية مسبقة
- "بدء القصة بدون صورة" option متاح — لا إجبار
- Mode Select يمنع القيم الفارغة

**مشاكل:**
1. **لا تحذير قبل مغادرة القصة:** إذا ضغط المستخدم Back من OS أثناء مشهد، يفقد تقدمه بصمت
2. **الـ VoiceToggle يُفعَّل بضغطة واحدة** في صفحة Landing — لا تأكيد قبل تشغيل الميكروفون

**الإصلاح:**
```
1. على Back gesture أثناء القصة:
   Alert: "ستفقد هذا المشهد إذا رجعت. متأكد؟"
   [رجوع] [أكمل القصة]

2. أضف tooltip/hint أول مرة: 
   "سيحتاج تفعيل الراوية الوصول للميكروفون"
   → يُقلل المفاجأة
```

---

### H6 — التعرف لا التذكر (Recognition Rather Than Recall)

**الدرجة: 5 / 5 ★★★★★**

**ما يعمل بشكل مثالي:**
- أيقونات Emoji للمشاعر — كونية، فورية الفهم
- ألوان مميّزة لكل نوع NarrationBlock (أبيض/ذهبي/سماوي)
- SceneProgress "المشهد ٢ من ٥" — سياق دائماً مرئي
- AudioHUD يُظهر الحالة الحالية بصريًا (لون + نص)

**ملاحظة:**
النظام يُريك ما تحتاج رؤيته في اللحظة المناسبة. لا تتذكر، بل تتعرف.

---

### H7 — المرونة والكفاءة (Flexibility and Efficiency of Use)

**الدرجة: 3 / 5 ★★★☆☆**

**ما يعمل:**
- اختيار مباشر للمشاعر (ضغطة واحدة = مشهد)
- الـ Mode Selector للمستخدم المتمرس

**مشاكل:**
1. **لا shortcuts للمستخدمين المتمرسين:** المستخدم العائد يمر بنفس الـ Onboarding (حتى مع تخطي)
2. **لا "آخر قصة":** لا يمكن استئناف قصة سابقة أو الاطلاع على تاريخ
3. **Judge Mode مخفي في Select:** المحكمون يجب أن يجدوه فورياً

**الإصلاح:**
```
للـ Returning User:
• أضف "Welcome Back" card في Emotion Picker باعتماد:
  - "قصتك الأخيرة: [عنوان]" مع زر "تكملة"
  
للـ Power Users:
• Shake gesture → Options menu سريع
• Judge Mode: أضف زر "⚖️ Judge Mode" بارز في أعلى الـ Picker
```

---

### H8 — الجماليات والتصميم المُبسَّط (Aesthetic and Minimalist Design)

**الدرجة: 5 / 5 ★★★★★**

**تقييم استثنائي — ونادر:**

> مرايا تفهم مبدأ Dieter Rams "أقل لكن أفضل" عبر:
- الشاشة الرئيسية فارغة في معظمها — الصمت البصري قرار مقصود
- لا tab bars، لا menus، لا notifications
- نص السرد وحده على أغلب الشاشة — لا شيء يُنافسه
- الـ HUD صغير ومخفي، لا يسرق الانتباه

**الإنجاز الحقيقي:** مقاومة إغراء إضافة features لمجرد "الكمال الوظيفي". هذا أصعب من البناء.

---

### H9 — مساعدة المستخدمين على التعرف والتشخيص والتعافي من الأخطاء (Help Users Recognize, Diagnose, and Recover from Errors)

**الدرجة: 3 / 5 ★★★☆☆**

**ما يعمل:**
- "توقف الخيط، نُعيد نسجه" — نبرة إنسانية لا تقنية ✅
- زرا "حاول مجدداً" و"اختر مشاعراً أخرى" — خيارات واضحة

**مشاكل:**
1. **Toast رسائل قصيرة جداً** — بعض الأخطاء (حجم الملف، صيغة خاطئة) تحتاج شرح أوضح
2. **لا حالة Offline مُصممة للشاشة الرئيسية** — الـ Toast لا يكفي لوضع قطع الإنترنت الكامل
3. **"Error 503" قد يظهر في الـ Console للمطور** ويصل للمستخدم في بعض الحالات

**الإصلاح:**
```
حالة Offline الكاملة:
┌──────────────────────┐
│   ◌  مرايا في صمت   │
│                      │
│  الاتصال مقطوع.      │
│  عِد حين يعود النبض  │  ← شاعرية بدلاً من "No Internet"
│                      │
│  [حاول مجدداً]       │
└──────────────────────┘
→ مع Animation هادئة (نبضات خفيفة)
```

### H10 — المساعدة والتوثيق (Help and Documentation)

**الدرجة: 4 / 5 ★★★★☆**

**ما يعمل:**
- الـ Onboarding يُغني عن الـ Help في معظم الحالات ✅
- Tooltips على الـ HUD buttons

**مشاكل:**
1. **لا Contextual Help داخل القصة:** المستخدم الجديد قد لا يفهم الفرق بين "سرد" و"تأمل" و"بصري"
2. **لا FAQ أو About** قابل للوصول بسهولة

**الإصلاح:**
```
أضف Tooltip عند أول ظهور لكل نوع NarrationBlock:
  NarrationBlock أول مرة: 
  "💡 هذا سرد — صوت الراوي يُكمل القصة"
  [فهمت ✓]  (يختفي إلى الأبد بعد التأكيد)
```

## القسم الثاني: التحليل البصري العميق

### أ. التسلسل الهرمي البصري

**الدرجة الإجمالية: 4.5 / 5**

#### ما هو ممتاز:

```
شاشة القصة — هرمية واضحة:

المستوى 1 (dominant): نص السرد — الأكبر والأبرز
المستوى 2 (secondary): أزرار الخيار — تحت النص مباشرة
المستوى 3 (tertiary): SceneProgress — صغير ومتراجع
المستوى 4 (invisible): HUD — يُرى فقط حين تبحث عنه
```

هذا **هرم مثالي** — العين تذهب للمحتوى أولاً، ثم للإجراء، ثم للسياق.

#### المشكلة الوحيدة:

```
في شاشة Emotion Picker:
المشكلة: العنوان "مرايا" والشبكة "٦ بطاقات" في منافسة بصرية.
لا يوجد عنصر dominant واضح.

الإصلاح:
• كبّر "مرايا" إلى 64px+ (hero text حقيقي)
• أضف فراغ كبير (80px) بينه وبين البطاقات
• البطاقات تظل متساوية (لأن لا قرار مُسبق)
```

---

### ب. الطباعة

**الدرجة الإجمالية: 4 / 5**

```
النقاط الإيجابية:
✅ Cairo للعربية — اختيار صحيح (ارتفاع السطر الكافي للعربية)
✅ clamp() للأحجام — تكيّف سلس
✅ نص السرد 1.85 line-height — مريح للقراءة العربية

المشاكل:
```

**المشكلة 1: قيم مُختلفة لنفس المستوى**

```css
/* الـ Spec يقول Title3 = 1.3rem min */
/* لكن في App.css الفعلي: */
.scene-progress { font-size: 0.9rem; }          /* Caption هنا؟ */
.emotion-picker__mode-label { font-size: 0.85rem; } /* أصغر من Caption */
.space-upload__title { font-size: 2rem; }       /* Title1؟ موحّد؟ */
.narration-text__label { font-size: 0.82rem; }  /* Overline؟ لكن لا uppercase */

→ لا يوجد ربط واضح بين Code و Type Scale المُعتمد
```

**الإصلاح:**
```css
/* طبّق scale بشكل صارم وثائقي */
.scene-progress-text  { font-size: var(--type-caption-size); }  /* 0.75–0.85rem */
.section-label        { font-size: var(--type-overline-size); text-transform: uppercase; }
.upload-title         { font-size: var(--type-title2-size); }
```

**المشكلة 2: Overline بدون uppercase في أماكن**

```
NarrationBlock labels: "📖 سرد" / "👁 بصري" / "🔮 تأمل"
الـ Spec يقول Overline = UPPERCASE
لكن نص العربية بـ UPPERCASE يبدو خاطئاً.

الحل الصحيح:
للعناوين اللاتينية: uppercase (NARRATION / VISUAL / REFLECTION)
للعناوين العربية: SemiBold + لون فقط، بدون uppercase
```

### ج. الألوان

**الدرجة الإجمالية: 4.5 / 5**

#### التقييم:

```
✅ قوي جداً:
• الـ Void Black (#030305) بدلاً من #000 = قرار صحيح (دفء خفي)
• color-mix() لتوهج البطاقات = حداثي ومرن
• تمييز ألوان 3 أنواع NarrationBlock (أبيض/ذهبي/سماوي) = ذكي

⚠️ مشكلة واحدة مهمة:
لوحة المشاعر تستخدم 6 ألوان مميزة — في تجربة واحدة.
الخطر: إذا ظهرت أكثر من 3 منها في نفس اللحظة، الشاشة ستُصبح مشوشة بصرياً.
```

**الإصلاح:**
```
قاعدة: في أي شاشة واحدة، لا يزيد عن لونَي مشاعر.
• EmotionPicker: الستة تظهر معاً لكن مُدمَّجة وخافتة → مقبول
• SceneView: لون واحد فقط (للمشاعر المُختار) → ✅
• مضمون: لا تُدمج Golden NarrationBlock داخل Joy-colored Scene
```

#### نسبة التباين — ملاحظة حرجة:

```
White 30% (--text-disabled) على #030305 = 6.3:1
→ يجتاز AA Large (≥ 3:1)، لكن يفشل AA Small (≥ 4.5:1)

الحالات المُستخدمة فيها White 30%:
• disabled buttons → مقبول (لا يُتوقع قراءة نص)
• BUT: إذا استُخدم في placeholder text → مشكلة WCAG

الإصلاح:
• White 40% (opacity) = 8.3:1 → يجتاز AA Small
• استبدل --text-disabled من 0.30 → 0.40 للنصوص فقط
```

### د. سهولة الاستخدام

**الدرجة الإجمالية: 3.5 / 5**

#### ما يعمل:
- Tap targets كافية (بطاقات 140px+ عرضاً)
- التدفق الخطي يُبسّط التعلم
- Language switching واضح

#### ما يحتاج عمل:

**مشكلة 1: Audio HUD ضيق جداً (حرج)**

```css
/* الوضع الحالي */
.audio-hud__btn {
  min-width: 170px;
  padding: 0.45rem 0.7rem; /* ≈ 7.2px vertical */
}
/* Touch target effective height: ~28px --- أقل من الحد الأدنى Apple (44pt) */
```

**إصلاح فوري:**
```css
.audio-hud__btn {
  min-height: 44px; /* Apple HIG minimum */
  padding: 0.6rem 0.9rem;
}
```

**مشكلة 2: ChoiceButtons تظهر فقط بعد اكتمال الكتابة**

```
تجربة المستخدم المُتعجّل:
• يريد الضغط قبل انتهاء النص → الأزرار مخفية
• يضغط على النص → لا رد فعل
• يشعر بـ "stuck"

الإصلاح:
Tap على النص السردي أثناء الكتابة → يُسرّع إلى النهاية فوراً
(Skip animation, show text immediately + أزرار)
Haptic: .light
```

### هـ. الاتساق الاستراتيجي

**الدرجة الإجمالية: 4.5 / 5**

```
التوافق بين الاستراتيجية والتنفيذ:

الاستراتيجية: "المرآة تعكس"  → التصميم: لا يفرض، لا يُخبر
✅ بطاقات المشاعر = مرآة للداخل
✅ الصور المُولَّدة = انعكاس بصري للمشاعر

"صوت الراوية: ON/OFF" → كلمات تقنية في عالم شاعري
الإصلاح:
"الراوية تتحدث / في صمت" بدل ON/OFF
```

## القسم الثالث: الحمل المعرفي

### تحليل Cognitive Load

```
Emotion Picker — حمل منخفض ✅
  • 6 خيارات فقط (ضمن 7±2 Miller's Law)
  • بصري فوري، لا قراءة مطوّلة
  • decision hierarchy واضح

Settings Sheet — حمل متوسط ⚠️
  • 5 إعدادات في نفس الوقت
  • Segmented + Toggle + Slider + 2 Disclosure = أنواع متعددة
  • قد يُربك المستخدم أول مرة

Scene View — حمل منخفض في الأمثل ✅
  • نص واحد في وقت واحد
  • زرا خيار فقط (لا أكثر)
  
لكن عند تراكم 3 NarrationBlocks — حمل مرتفع ⚠️
  • نص سرد + نص بصري + نص تأمل في نفس الوقت
  • 3 حدود ألوان مختلفة + 3 تسميات
  → يُشتّت التركيز عن الجوهر
```

**الإصلاح:**
```
قاعدة: اعرض block واحد في كل مرة، بدون تراكم.
الترتيب: السرد → يختفي → البصري يظهر → يختفي → التأمل
بدلاً من: ثلاثتهم في نفس اللحظة

استثناء: Transcript جانبي يُراكم (للقراءة الاختيارية)
```

## القسم الرابع: وضوح التفاعل

```
ما هو واضح جداً:
✅ بطاقات المشاعر = tap-to-select (بصري فوري)
✅ أزرار الخيار السردي = tap واضح (arrow + text)
✅ upload zone = drag or tap (نص واضح)

ما هو غامض:
⚠️ النص السردي قابل للضغط (Tap) → لكن لا يوجد Affordance
   المستخدم الجديد لن يعرف أن الضغط يُسرع الكتابة

⚠️ Live Redirect Bar = يُغيّر المشاعر مباشرة؟ أم ينتقل لصفحة جديدة؟
   غير واضح من الـ UI الحالي

⚠️ Transcript (Judge Mode) = لا يوجد handle أو scroll indicator
   المستخدم لا يعلم إذا كان يمكنه التمرير
```

**الإصلاحات:**
```

   Tooltip على أول ظهور
```

### الاتجاه البديل الأول: "القصيدة المرئية" (Visual Poetry)

**الفلسفة:** بدلاً من Glass cards وبلوكات، النص يتدفق مباشرة على الصورة.

```
التحول الجذري:
• لا NarrationBlock منفصلة
• النص يُكتب مباشرة على الصورة بـ text-shadow cinematic
• كل جملة تظهر وتتلاشى وحدها (fade per line)
• لا حدود، لا خلفيات، لا labels

تخطيط شاشة القصة:
┌───────────────────────────────┐
│                               │
│   [صورة سينمائية كاملة]      │
│                               │
│   وفي تلك اللحظة،            │
│   كان القمر يبدو              │
│   أقرب من أيّ وقت مضى...    │
│                               │
│      ↓ (اسحب للأسفل)         │
│                               │
└───────────────────────────────┘
→ الخيارات تظهر بـ Swipe Up
```

**المزايا:**
- أكثر غمراً وسينمائية
- يُشبه تجربة Instagram Stories المُدمجة بالـ AI
- يُقلل التعقيد البصري بشكل جذري

**العيوب:**
- قد يصعب القراءة على أنواع معينة من الصور
- يفقد تمييز أنواع السرد الثلاثة (سرد/بصري/تأمل)
- يتطلب قواعد text-shadow أكثر تعقيداً

**متى تختاره:**
إذا كانت الأولوية الغمر المطلق على حساب وضوح البنية.

---

### الاتجاه البديل الثاني: "المسرح التفاعلي" (Interactive Theatre)

**الفلسفة:** شاشة القصة تُشبه مسرحاً — الصورة "خشبة مسرح"، والنص "ديكور"، والخيارات "نص المسرحية".

```
التخطيط:
┌─────────────────────────────────────────────┐
│  ═══════════════════════════════════════    │ ← Top Progress (thin line full width)
│                                             │
│                                             │
│   [صورة مولّدة — crop يملأ 60% الشاشة]   │
│         مع vignette gradient                │
│                                             │
│═════════════════════════════════════════════│ ← قسم أسفل مُميَّز بـ gradient separator
│                                             │
│  ╔═════════════════════════════════════╗   │
│  ║                                     ║   │
│  ║  النص يظهر word-by-word (لا char)  ║   │
│  ║  فقرة أعمق + اقتباس مُميَّز        ║   │
│  ║                                     ║   │
│  ╚═════════════════════════════════════╝   │
│                                             │
│  ←  [خيار 1]        [خيار 2]  →            │ ← Left/Right swipe cards
│                                             │
└─────────────────────────────────────────────┘
```

**الابتكار الرئيسي — Swipe Choices:**
```
بدلاً من ضغط زر:
• Swipe Right → الخيار 1
• Swipe Left  → الخيار 2
→ يُشبه Tinder لكن للقصص
→ haptic.medium عند activation
→ الكارت يتوسع/يتمدد لتعبّر عن الخيار
```

**المزايا:**
- تفاعل جسدي (embodied interaction) = أعمق انخراطاً
- يُميّز الخيارات بشكل مكاني (يمين ≠ يسار)
- الـ Swipe gesture أسرع من الـ Tap

**العيوب:**
- قد يَخلط RTL بـ meaning of left/right choices
- يحتاج في الـ Onboarding شرحاً للـ gesture
- أقل وضوحاً للمستخدم المسنّ أو غير المُتمرّس

**متى تختاره:**
إذا أردت أن تجعل التفاعل جزءاً من المشاعر (الاتجاه = المعنى).
